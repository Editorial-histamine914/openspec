# Changelog

All notable changes to OpenSpec will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.2.3] - 2026-03-13

### Fixed
- Fixed a bug where `openspec status` and `openspec clean` would fail to recognize existing context files due to strict case matching in the banner check.
- Updated test suite assertions to correctly match the latest banner format.

## [0.2.2] - 2026-03-13

### Added
- Internationalization: Added multi-language support to README (Chinese Simplified `zh-CN` and Portuguese Brazilian `pt-BR`).
- Enhanced viral marketing features in README and project metadata.
- Enabled GitHub Sponsors button via `FUNDING.yml`.

### Changed
- Improved the "Generated by OpenSpec" banner with a more professional design and helpful links.
- Updated README with clearer "Stop Context Drift" positioning and benefit-driven feature lists.

## [0.2.1] - 2026-03-13

### Added
- **Claude Code Plugin & Marketplace support:** Truly Zero-Init workflow.
- `openspec analyze` command for deep codebase analysis and AI-powered rule generation.
- `openspec install` command to install the Claude Code plugin locally.

## [0.1.0] - 2026-03-12

### Added

- Core transpiler engine with module discovery and compilation
- 7 target outputs: Claude Code, Cursor, Gemini, GitHub Copilot, Aider, OpenAI Codex, Windsurf
- YAML frontmatter support: `name`, `description`, `priority`, `targets`, `excludeTargets`, `tags`, `globs`
- Per-target renderers with tool-specific formatting
- CLI commands: `init`, `sync`, `watch`, `status`, `hooks`, `clean`
- File watcher with debounced auto-sync (`openspec watch`)
- Git pre-commit hook integration (`openspec hooks`)
- YAML and JSON config file support with multiple search paths
- `openspec init` scaffolding with example modules
- `openspec clean` to remove only openspec-managed generated files
